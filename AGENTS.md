# Bootstrap Instructions

## Current phase

This repository is currently in the specification and architecture phase.

## Mandatory rules

- Work only inside this repository.
- Do not access files outside the workspace.
- Do not generate production code before completing and validating SDD.
- Execute only Phase 0 and Phase 1 unless explicitly instructed otherwise.
- Do not install dependencies during Phase 0 or Phase 1.
- Do not create or expose API keys.
- Do not modify global system configuration.
- Do not push, merge, deploy, publish packages, or modify remote repositories.
- Do not execute destructive commands.
- Do not use shell=True.
- Do not modify main through automated commits.
- Document assumptions instead of silently inventing requirements.
- Report commands executed and their real results.
- Never claim that a command, test, or validation passed unless it was executed.
- Prefer simple architecture and justified design patterns.
- Stop after completing the requested phase.

## Required workflow

1. Inspect the repository.
2. Create the SDD specification.
3. Validate consistency and traceability.
4. Present findings and risks.
5. Stop and wait for an explicit phase-approval instruction.
